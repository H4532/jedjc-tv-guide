Sales publishing workflow

  1. Sign in using the Sales team member's approved GitHub account.
  2. Create and upload two approved images with the same base name: campaign-desktop.webp and campaign-mobile.webp.
  3. Open the matching data.json editor.
  4. Copy the existing template item and update both English and Arabic content.
  5. Set imageDesktop and imageMobile to the two uploaded file paths.
  6. Set active to true, then select Commit changes.
  7. Wait approximately one to three minutes and test the page on both phone and desktop.
Access requirement: the fixed page login only hides the management interface from casual visitors. Each Sales team member still needs an individual GitHub account added as a collaborator to H4532/jedjc-tv-guide before they can upload or publish content.
